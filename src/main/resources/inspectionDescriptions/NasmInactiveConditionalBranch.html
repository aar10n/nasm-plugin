Reports code inside inactive preprocessor conditional branches.

The inspection evaluates %ifdef, %ifndef, %if, and other conditional directives based on:

  • Macro definitions from project settings (command-line defines)
  • Macro definitions in source files (%define, %assign)
  • Macro definitions in included files

Code determined to be in an inactive branch is greyed out, similar to unused symbols, and excluded from code navigation and reference resolution.

Example:

  %ifdef DEBUG
  ; This code is active if DEBUG is defined in settings
  call debug_log
  %else
  ; This code is inactive if DEBUG is defined (greyed out)
  nop
  %endif

Configuration: Define command-line macros in Settings → Languages & Frameworks → NASM Assembly → Preprocessor Macros.

Disable this inspection if you want to see all conditional branches equally, for example when working on cross-platform assembly code where you need to see both platform-specific implementations.
